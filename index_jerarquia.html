Jerarquia del modelo OSI: Protocolos

Por: Brandon Steven P. Giraldo

ING. Sistemas - 1er semestre



El modelo OSI (Open Systems Interconnection) es un marco conceptual que divide las funciones de una red de computadoras en siete capas. Cada capa se encarga de tareas específicas, y la comunicación entre las capas permite la transferencia de datos entre dispositivos en una red. Desde la capa física que trata con cables y señales eléctricas hasta la capa de aplicación que maneja la interacción con el usuario, el modelo OSI proporciona una estructura organizada para entender y diseñar sistemas de red. Este enfoque modular facilita la interoperabilidad entre diferentes fabricantes y tecnologías.


1. Capa Física (Physical Layer)

Se encarga de la transmisión física de bits a través de medios de comunicación como cables, señales eléctricas o inalámbricas. Define las características eléctricas, mecánicas y funcionales de los dispositivos de red.

Ethernet

El protocolo Ethernet es un conjunto de reglas y estándares que define cómo se transmiten los datos en una red de área local (LAN).

USB

USB significa Universal Serial Bus. Es una interfaz de conexión estándar que permite la conexión y comunicación de dispositivos con una computadora u otros dispositivos electrónicos.

BLUETOOTH

Bluetooth es una tecnología inalámbrica que permite la transferencia de datos entre dispositivos electrónicos a corta distancia. Se utiliza comúnmente para conectar dispositivos como teléfonos móviles, auriculares, altavoces y teclados de forma inalámbrica.





2. Capa de Enlace de Datos (Data Link Layer)

Proporciona la transferencia fiable de datos entre nodos directamente conectados. Se encarga de la detección y corrección de errores en la capa física y controla el acceso al medio.

Ethernet

El protocolo Ethernet es un conjunto de reglas y estándares que define cómo se transmiten los datos en una red de área local (LAN).

PPP

(Point-to-Point Protocol) es un protocolo de red utilizado para establecer una conexión de red punto a punto. Se utiliza comúnmente en conexiones de acceso remoto, como la conexión a Internet a través de módems. PPP proporciona autenticación, compresión y encriptación de datos para una comunicación segura y confiable.

HDLC

(High-Level Data Link Control) es un protocolo de enlace de datos de nivel superior utilizado para la transmisión confiable de datos entre dispositivos en una red. HDLC se utiliza comúnmente en conexiones de red de área amplia (WAN) y proporciona funciones como detección y corrección de errores, control de flujo y sincronización de datos.



3. Capa de Red (Network Layer)

Se ocupa del enrutamiento de datos a través de la red. Determina la mejor ruta para enviar datos desde el origen hasta el destino, independientemente de la topología física de la red.

IPV4

(Internet Protocol version 4) es la cuarta versión del Protocolo de Internet. Es el protocolo de red más utilizado en Internet y se utiliza para asignar direcciones IP a dispositivos en una red. IPv4 utiliza direcciones IP de 32 bits, lo que permite un total de aproximadamente 4.3 millones de direcciones unicas.

IPV6

(Internet Protocol version 6) es la sexta versión del Protocolo de Internet. Fue desarrollado para reemplazar a IPv4 debido a la escasez de direcciones IP disponibles en este último. IPv6 utiliza direcciones IP de 128 bits, lo que proporciona un espacio de direcciones mucho más grande en comparación con IPv4.

ICPM

(Internet Control Message Protocol) es un protocolo de control y diagnóstico utilizado en redes IP. ICMP se utiliza para enviar mensajes de error, notificaciones y solicitudes de prueba entre dispositivos en una red. Por ejemplo, se utiliza para enviar mensajes de "ping" para verificar la conectividad entre dos dispositivos.

DHCP

(Dynamic Host Configuration Protocol) es un protocolo de red utilizado para asignar automáticamente direcciones IP y otros parámetros de configuración a dispositivos en una red. DHCP permite que los dispositivos obtengan una dirección IP única cada vez que se conectan a la red, evitando conflictos de direcciones IP.



4. Capa de Transporte (Transport Layer)

Ofrece servicios de transporte de extremo a extremo para la transferencia de datos entre aplicaciones. Se encarga del control de flujo, la segmentación y el reensamblaje de datos, y la corrección de errores.

TCP

(Transmission Control Protocol) es un protocolo de transporte utilizado en redes de computadoras. Proporciona una comunicación confiable y orientada a la conexión entre dispositivos en una red. TCP divide los datos en paquetes y los envía de manera secuencial, asegurándose de que lleguen correctamente y en el orden correcto.

UDP

(User Datagram Protocol) es un protocolo de transporte utilizado en redes de computadoras. A diferencia de TCP, UDP es un protocolo de conexión no confiable y orientado a datagramas. Esto significa que no se garantiza la entrega de paquetes ni el orden en que llegan. UDP se utiliza para aplicaciones que requieren una comunicación rápida y eficiente, como transmisiones en tiempo real, juegos en línea y servicios de transmisión de video.

SCTP

(Stream Control Transmission Protocol) es un protocolo de transporte utilizado en redes de computadoras. A diferencia de TCP y UDP, SCTP está diseñado para proporcionar una comunicación confiable, ordenada y orientada a la conexión. SCTP ofrece características como la entrega confiable de mensajes, el control de congestión y la detección de errores.



5. Capa de Sesión (Session Layer)

Establece, mantiene y termina las sesiones de comunicación entre aplicaciones. Controla el diálogo y sincronización entre los dispositivos.

NetBIOS

(Network Basic Input/Output System) es un conjunto de protocolos de red utilizado en sistemas operativos Windows para permitir la comunicación entre dispositivos en una red local. NetBIOS proporciona servicios como resolución de nombres, compartición de archivos e impresión, y descubrimiento de recursos en la red.

RCP

RCP puede referirse a dos conceptos diferentes: 1. RCP (Remote Copy Protocol) es un protocolo utilizado para copiar archivos de forma remota entre computadoras en una red. RCP es similar al comando "cp" en sistemas Unix y Linux, pero permite la copia de archivos entre sistemas diferentes. 2. RCP (Remote Procedure Call) es un mecanismo utilizado en la programación distribuida para permitir que un programa en una computadora llame a un procedimiento o función en otra computadora de forma transparente. RCP permite la comunicación entre procesos en diferentes sistemas, como si estuvieran ejecutándose localmente.

PPTP

(Point-to-Point Tunneling Protocol) es un protocolo de red utilizado para establecer conexiones VPN (Virtual Private Network) entre dispositivos.



6. Capa de Presentación (Presentation Layer)

Se encarga de la representación y la traducción de datos. Realiza funciones como la compresión, cifrado y conversión de formatos para garantizar la compatibilidad entre sistemas heterogéneos.

SSL/TLS

(Secure Sockets Layer) y TLS (Transport Layer Security) son protocolos de seguridad criptográfica utilizados para establecer conexiones seguras en Internet, como HTTPS.

JPEG

(Joint Photographic Experts Group) es un formato de compresión de imágenes ampliamente utilizado para almacenar fotografías digitales.

GIF

(Graphics Interchange Format) es un formato de imagen ampliamente utilizado para animaciones y gráficos en movimiento. Es compatible con la compresión sin pérdida de calidad y admite una paleta de hasta 256 colores.



7. Capa de Aplicación (Application Layer)

Proporciona servicios de red directamente a las aplicaciones y actúa como interfaz entre el software de aplicación y la red. Incluye protocolos para servicios como el correo electrónico, la transferencia de archivos y la navegación web.

HTTP

(Hypertext Transfer Protocol) es un protocolo de comunicación utilizado para transferir datos a través de Internet. Es el protocolo principal utilizado para la comunicación entre un cliente (navegador web) y un servidor web.

SMTP

(Simple Mail Transfer Protocol) es un protocolo de red utilizado para el envío de correo electrónico. Es responsable de la transferencia y entrega de mensajes de correo electrónico entre servidores de correo electrónico.

FTP

(File Transfer Protocol) es un protocolo de red utilizado para la transferencia de archivos entre un cliente y un servidor en una red. Permite la subida y descarga de archivos de manera eficiente y segura.

IMAP

(Internet Message Access Protocol) es un protocolo de correo electrónico que permite a los usuarios acceder y administrar sus correos electrónicos almacenados en un servidor remoto. A diferencia del protocolo POP, IMAP permite mantener los correos electrónicos en el servidor y sincronizar cambios entre dispositivos.

POP3

POP3 (Post Office Protocol version 3) es un protocolo de correo electrónico utilizado para recuperar mensajes de correo electrónico desde un servidor remoto a un cliente de correo. A diferencia de IMAP, POP3 descarga los correos electrónicos del servidor y los almacena localmente en el dispositivo del usuario.
